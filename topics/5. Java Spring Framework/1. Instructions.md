### **SESSION 1: SPRING CORE & FUNDAMENTALS**

**Goal:** Understand the philosophy of Spring, Inversion of Control (IoC), and Dependency Injection (DI).

#### **1. Scope (Key Knowledge Areas)**

- **a) Spring Framework Overview:**
  - Spring Ecosystem & Benefits.
  - Spring vs Spring Boot (Why Spring Boot?).
  - Auto-configuration concept (The "Magic").

- **b) Core Concepts (IoC Container):**
  - **Inversion of Control (IoC):** The shift of control.
  - **Dependency Injection (DI):** Types (Constructor vs Field vs Setter).
  - **Bean & ApplicationContext:** What is a Bean? Container role.
  - **Bean Lifecycle:** Creation, Initialization, Destruction hooks.
  - **Configuration:** `@Component` (Stereotypes: `@Service`, `@Repository`), `@Configuration`, `@Bean`.

#### **2. Instruction: How to leverage AI/Web**

Sample prompts:

- **IoC:** "Explain Inversion of Control (IoC) using a 'Car and Engine' analogy. Why is it better than `new Engine()`?"
- **Spring Boot:** "How does Spring Boot differ from legacy Spring Framework? What is 'Opinionated Defaults'?"
- **DI Types:** "Compare Constructor Injection vs Field Injection (`@Autowired`). Why is Constructor Injection recommended?"
- **Beans:** "What is the difference between `@Component` and `@Bean`? When to use which?"
- **Scopes:** "What are Bean Scopes (Singleton, Prototype, Request)? What happens if I inject a Prototype into a Singleton?"
- **Lifecycle:** "Visualize the Spring Bean Lifecycle. When does `@PostConstruct` run?"

### **SESSION 3: SPRING WEB MVC & REST API**

**Goal:** Build modern RESTful APls using Spring Web.

#### **1. Scope**

- **a) Spring Web Architecture:**
  - **DispatcherServlet:** The Front Controller pattern.
  - **Request Flow:** Controller → Service → Repository.
  - **Handler Mapping:** Resolution mechanism.

- **b) REST Implementation:**
  - **Annotations:** `@RestController` vs `@Controller`.
  - **Mappings:** `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`.
  - **Inputs:** `@PathVariable` (Resource ID) vs `@RequestParam` (Filters).
  - **DTOs:** Request DTO vs Response DTO (Decoupling entity).
  - **HTTP:** Proper Status Codes (`200 OK`, `201 Created`, `400 Bad Req`, `404 Not Found`).

- **c) Layer Separation:**
  - Controller layer responsibility (Routing, Validation).
  - Service layer responsibility (Business Logic).

- **d) API Testing Tools:**
  - **cURL:** CLI basics (`-X POST`, `-H`, `-d`).
  - **Postman:** [postman.com](https://www.postman.com/)

#### **2. Instruction: How to leverage AI/Web**

- **Architecture:** "Trace the flow of an HTTP Request in Spring Boot from the Browser to the Controller method."
- **REST:** "What are the rules of RESTful API design? Why should I return `201 Created` for POST?"
- **Params:** "When should I use Query Parameters (`?name=`) vs Path Variables (`/users/1`)? Give examples."
- **DTO:** "Why use DTOs (Data Transfer Objects) instead of returning JPA Entities directly regarding Security and Versioning?"
- **Idempotency:** "Which HTTP methods are Idempotent? Why does it matter?"
- **Tools:** "Show me a cURL command to POST a JSON object to `/api/v1/students`."
- **Automation:** "How to write a simple test in Postman to verify status code is 201?"

_Action:_ Complete **Session 3 Labs** in `2. Mini Labs.md`.

### **SESSION 4: DATA PERSISTENCE (JPA & HIBERNATE)**

**Goal:** Connect to Databases and manage data using Object-Relational Mapping (ORM).

#### **1. Scope**

- **a) JPA & Spring Data JPA:**
  - Concepts: JPA (Spec) vs Hibernate (Impl) vs Spring Data JPA (Abstraction).
  - **Entity Mapping:** `@Entity`, `@Table`, `@Id`, `@GeneratedValue`, `@Column`.
  - **Relationships:** One-to-Many, Many-to-One `@JoinColumn`.

- **b) Operations & Validation:**
  - **Repository Pattern:** `JpaRepository<T, ID>`, Derived Query Methods (`findByEmail`).
  - **Transactions:** `@Transactional` boundaries, Rollback rules.
  - **Pagination:** `Pageable`, `PageRequest`.
  - **Validation:** Bean Validation (`@NotNull`, `@Size`, `@Email`) & Exception Handling (`@ControllerAdvice`).

#### **2. Instruction: How to leverage AI/Web**

- **Ecosystem:** "Explain the relationship between JPA, Hibernate, and Spring Data JPA."
- **Entity:** "How to map a Java Class to a SQL Table? What does `@Transient` do?"
- **Repository:** "How does Spring Data JPA generate SQL automatically from method names like `findByNameAndAge`?"
- **Transactions:** "What happens if an Exception is thrown inside a `@Transactional` method? Checked vs Unchecked exception rollback?"
- **N+1 Problem:** "What is the N+1 Select Problem in Hibernate? How does `JOIN FETCH` solve it?"
- **Validation:** "How to validate a Request Body automatically using `@Valid`? Where do errors go?"

_Action:_ Complete **Session 4 Labs** in `2. Mini Labs.md`.
